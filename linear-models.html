Linear models

Linear, logistic, and softmax regression — three faces of the same underlying machinery. One loss type per task, one gradient pattern for all.

1. Supervised learning setup

A supervised dataset:

$$\mathcal{D} = \{(\mathbf{x}^{(1)}, y^{(1)}), \ldots, (\mathbf{x}^{(N)}, y^{(N)})\}.$$
Where: $N$ = number of training examples; $\mathbf{x}^{(i)}$ = feature vector of the $i$-th example (column vector in $\mathbb{R}^d$); $y^{(i)}$ = target/label; the superscript $(i)$ is an index, not a power.

Goal: find a hypothesis $h: \mathcal{X} \to \mathcal{Y}$ that approximates an unknown function $f$. If $\mathcal{Y}$ is continuous → regression; if discrete → classification.

2. Master decision table — which model, loss, activation?

This single table tells you everything. Pick the row matching your problem.

Task Model Output activation Loss function Target shape
Regression (continuous) Linear regression Identity (linear) SSE / MSE $y \in \mathbb{R}$
Binary classification Logistic regression Sigmoid Binary Cross Entropy $y \in \{0,1\}$
Multi-class (exactly one class) Softmax regression Softmax Categorical Cross Entropy $\mathbf{y} \in \{0,1\}^K$ one-hot
Multi-label (multiple classes possible) $K$ logistic regressions Sigmoid (each) Sum of $K$ BCEs $\mathbf{y} \in \{0,1\}^K$ (not one-hot)
Multi-class via OVR $K$ binary logistic models Sigmoid (each) BCE per model $y \in \{1,\ldots,K\}$
Quick decision guide
  • Output is a number (price, points, time) → Linear regression.
  • Output is yes/no, spam/not, win/lose → Logistic regression.
  • Output is exactly one of $K$ categories (digit recognition, player position) → Softmax.
  • Output can be multiple categories at once (movie genres, image tags) → Multi-label sigmoid.
  • Already have a logistic library and need multi-class? → OVR ($K$ binary models).

3. Linear regression

With the trick $x_0 = 1$ so the bias is absorbed into the weight vector:

Hypothesis $$h_{\mathbf{w}}(\mathbf{x}) = w_0 + w_1 x_1 + \cdots + w_d x_d = \sum_{i=0}^{d} w_i x_i = \mathbf{w}^\top \mathbf{x}.$$
Where: $\mathbf{x} = (x_0, x_1, \ldots, x_d)^\top$ with $x_0 = 1$ so the bias is just another weight; $\mathbf{w}$ = weight vector; $w_0$ = bias/intercept; $d$ = number of original features.

It's called linear because it's linear in the weights $w_i$, not in the features $x_i$. (You can use polynomial features and still have a "linear model".)

🏀 NBA — predicting points per game

Features: $x_1$ = minutes played, $x_2$ = field goal attempts (FGA), $x_3$ = free throw attempts (FTA).

After training: $\hat{\mathbf{w}} = (1.2, 0.3, 1.1, 0.88)$.

For a player who played 36 minutes with 20 FGA and 5 FTA:

$$\hat y = 1.2 + (0.3)(36) + (1.1)(20) + (0.88)(5) = 1.2 + 10.8 + 22.0 + 4.4 = 38.4 \text{ points}.$$

Each additional FGA raises expected points by 1.1.

Loss function (SSE / MSE)

Sum of Squared Errors $$L(\mathbf{w}) = \frac{1}{2}\sum_{i=1}^{N}\left(h_{\mathbf{w}}(\mathbf{x}^{(i)}) - y^{(i)}\right)^2.$$
Where: $h_{\mathbf{w}}(\mathbf{x}^{(i)}) - y^{(i)}$ = residual; we square so signs don't cancel and big errors get penalized more; the $\tfrac{1}{2}$ is for derivative neatness. MSE = $\tfrac{1}{N}$ × SSE.

4. The normal equation

Stack the design matrix $\mathbf{X}$ (each row = one example) and target vector $\mathbf{y}$:

$$L(\mathbf{w}) = \tfrac{1}{2}\mathbf{r}^\top\mathbf{r}, \quad \mathbf{r} = \mathbf{X}\mathbf{w} - \mathbf{y}, \quad \nabla_{\mathbf{w}} L = (\mathbf{X}\mathbf{w} - \mathbf{y})^\top \mathbf{X}.$$

Setting the gradient to zero gives the closed-form solution:

Normal equation $$\boxed{\;\mathbf{w}^{*} = (\mathbf{X}^\top \mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y}\;}$$

Worked example

Data $(x, y)$: $(1, 2), (2, 2), (3, 4)$. With $x_0 = 1$ prepended:

$$\mathbf{X} = \begin{pmatrix} 1 & 1 \\ 1 & 2 \\ 1 & 3 \end{pmatrix}, \quad \mathbf{y} = \begin{pmatrix} 2 \\ 2 \\ 4 \end{pmatrix}, \quad \mathbf{X}^\top\mathbf{X} = \begin{pmatrix} 3 & 6 \\ 6 & 14 \end{pmatrix}, \quad \mathbf{X}^\top\mathbf{y} = \begin{pmatrix} 8 \\ 18 \end{pmatrix}.$$

Determinant: $3 \cdot 14 - 6 \cdot 6 = 6$, so $(\mathbf{X}^\top\mathbf{X})^{-1} = \tfrac{1}{6}\begin{pmatrix} 14 & -6 \\ -6 & 3 \end{pmatrix}$. Then:

$$\mathbf{w}^{*} = \tfrac{1}{6}\begin{pmatrix} 14 \cdot 8 - 6 \cdot 18 \\ -6 \cdot 8 + 3 \cdot 18 \end{pmatrix} = \tfrac{1}{6}\begin{pmatrix} 4 \\ 6 \end{pmatrix} = \begin{pmatrix} 0.667 \\ 1 \end{pmatrix}.$$

So $h(x) = 0.667 + x$. Residuals $(-0.333, 0.667, -0.333)$, SSE $\approx 0.333$.

5. Regularization — L1 (Lasso) & L2 (Ridge)

Motivation: linear models with many features overfit by tying themselves to noise. Solution: add a penalty on the size of the weights.

Regularized loss $$J(\mathbf{w}) = \underbrace{\frac{1}{2N}\sum_{i=1}^{N}\left(h_{\mathbf{w}}(\mathbf{x}^{(i)}) - y^{(i)}\right)^2}_{\text{MSE}} + \underbrace{\lambda \cdot \Omega(\mathbf{w})}_{\text{regularization}}.$$
L1 — Lasso
$\Omega(\mathbf{w}) = \|\mathbf{w}\|_1 = \sum_j |w_j|$
L2 — Ridge
$\Omega(\mathbf{w}) = \|\mathbf{w}\|_2^2 = \sum_j w_j^2 = \mathbf{w}^\top\mathbf{w}$
Key properties (exam favorites)
  • If $\lambda = 0$ → $J(\mathbf{w})$ is not regularized (reduces to plain MSE).
  • If $\lambda$ is very large on L2 → the model underfits (not overfits). Weights forced toward 0, model too simple.
  • L1 induces feature selection — some $w_j$ become exactly 0, dropping those features entirely.
  • L2 shrinks all weights toward 0 but rarely makes them exactly zero.
GoalChoiceReason
Feature selectionL1 (Lasso)Forces weights to zero
StabilityL2 (Ridge)Shrinks all weights
General overfit preventionL2Smoother, more convex
🏀 NBA — feature selection with Lasso

Predict MVP score from 30 features (PPG, RPG, APG, FG%, 3P%, FT%, BPG, SPG, TOV, Win%, …). Apply Lasso with $\lambda = 0.1$.

After training, only ~6 features end up with $w_j \neq 0$ (PPG, RPG, APG, FG%, Win%, BPG). The other 24 are dropped automatically. With Ridge, all 30 keep small nonzero weights — useful when you believe every statistic contributes a little.

Fitting curve — under/overfit visualized

Underfitting ←— Good fit —→ Overfitting

  • Underfitting: model too simple (e.g., $M = 1$ polynomial). High bias, fails to capture pattern.
  • Overfitting: model too flexible (e.g., $M = 9$ on 10 data points). High variance, fits noise.

Connection to regularization: large $\lambda$ → simpler model → toward underfit. Small $\lambda$ → more flexible → toward overfit. Pick $\lambda$ via a validation set.

6. Gradient descent

Update rule (all linear models) $$\mathbf{w} := \mathbf{w} - \alpha \, (\nabla_{\mathbf{w}} L(\mathbf{w}))^\top.$$
Where: $\alpha$ = learning rate (small positive, e.g. 0.01); $\nabla_{\mathbf{w}}L$ = gradient (direction of steepest increase); we move opposite. Transpose because $\nabla L$ comes out as a row vector while $\mathbf{w}$ is a column.

The universal gradient pattern

For ALL linear models $$\nabla_{\mathbf{w}} L = \sum_i (\text{prediction}^{(i)} - \text{target}^{(i)}) \cdot \mathbf{x}^{(i)\top}.$$

Only the meaning of "prediction" changes:

  • Linear regression: $\nabla_{\mathbf{w}} L = (\mathbf{X}\mathbf{w} - \mathbf{y})^\top \mathbf{X}$
  • Logistic regression: $\nabla_{\mathbf{w}} L = (\text{Sig}(\mathbf{X}\mathbf{w}) - \mathbf{y})^\top \mathbf{X}$
  • Softmax regression: $\dfrac{\partial L}{\partial \mathbf{W}} = \sum_i (\mathbf{s}^{(i)} - \mathbf{y}^{(i)})\mathbf{x}^{(i)\top}$

With L2: add $2\lambda\mathbf{w}$ to the gradient. With L1: add $\lambda\,\text{sign}(\mathbf{w})$.

Variants

VariantUpdatesProperties
Batch GD1 update after scanning all $N$ examplesAccurate gradient, slow
Stochastic GD (SGD)1 update per exampleFast & noisy; noise can escape local minima
Minibatch GD1 update per ~10–20 examplesBest compromise — what's used in practice

Stopping criteria

  • Fixed number of iterations.
  • $\|\mathbf{w}' - \mathbf{w}\| \leq \varepsilon$ — small weight change.
  • $|L(\mathbf{w}') - L(\mathbf{w})| \leq \varepsilon$ — small loss change.
  • $\|\nabla L(\mathbf{w})\| \leq \varepsilon$ — gradient near zero.
Tutorial — stopping criterion

For $\mathbf{w} = (w_0, w_1, w_2)^\top$, a valid criterion is:

$$(w_0^{(k+1)} - w_0^{(k)})^2 + (w_1^{(k+1)} - w_1^{(k)})^2 + (w_2^{(k+1)} - w_2^{(k)})^2 \leq \varepsilon^2,$$

i.e., $\|\mathbf{w}^{(k+1)} - \mathbf{w}^{(k)}\|_2^2 \leq \varepsilon^2$.

Worked example — GD on $L(w) = w^2 - 3w + 1$

$L'(w) = 2w - 3$, minimum at $w^* = 1.5$. Start $w_0 = 0$, $\alpha = 0.1$:

7. Logistic regression

The sigmoid function

Sigmoid $$g(z) = \text{Sig}(z) = \frac{1}{1 + e^{-z}} = \frac{e^z}{e^z + 1}.$$

Properties:

  • $0 < \text{Sig}(z) < 1$ — maps any real to a probability.
  • $\text{Sig}(-z) = 1 - \text{Sig}(z)$.
  • $\text{Sig}'(z) = \text{Sig}(z)(1 - \text{Sig}(z))$ — the clean derivative that makes backprop nice.

Model & decision boundary

Logistic regression model $$h_{\mathbf{w}}(\mathbf{x}) = \text{Sig}(\mathbf{w}^\top\mathbf{x}), \qquad \log\frac{p}{1-p} = \mathbf{w}^\top\mathbf{x} \text{ (linear log-odds)}.$$

Decision rule: predict class 1 if $h \geq 0.5$, else class 0 (equivalent to $\mathbf{w}^\top\mathbf{x} \geq 0$).

Binary cross entropy (BCE)

BCE loss $$\text{NLL}(\mathbf{w}) = -\sum_{i=1}^{M}\left[y^{(i)}\log h_{\mathbf{w}}(\mathbf{x}^{(i)}) + (1-y^{(i)})\log(1 - h_{\mathbf{w}}(\mathbf{x}^{(i)}))\right].$$

When $y^{(i)} = 1$ only the left term contributes; when $y^{(i)} = 0$ only the right term. Minimizing NLL = maximizing log-likelihood under a Bernoulli model.

Gradient ascent vs descent — same answer

Ascent (maximize log-likelihood $\ell$):

$$w_j := w_j + \alpha \sum_{i=1}^{m}(y^{(i)} - h_{\mathbf{w}}(\mathbf{x}^{(i)}))x_j^{(i)}.$$

Descent (minimize NLL):

$$w_j := w_j - \alpha \sum_{i=1}^{m}(h_{\mathbf{w}}(\mathbf{x}^{(i)}) - y^{(i)})x_j^{(i)}.$$

Both produce the same update; the ± signs cancel. What is NOT correct: a single-example update without the $\sum_{i=1}^{m}$ when doing batch GD.

Note: there is no closed-form solution for logistic regression because of the nonlinear sigmoid — must use GD.

🏀 NBA — clutch shot success

Features: $x_1$ = distance to rim (ft); $x_2$ = time remaining (sec); $x_3$ = defender distance (ft).

Trained: $\hat{\mathbf{w}} = (2.0, -0.1, 0.05, 0.3)$. Damian Lillard shoots from $x_1 = 28$, $x_2 = 4$, $x_3 = 5$.

Step 1 — compute the logit:

$$z = 2.0 + (-0.1)(28) + (0.05)(4) + (0.3)(5) = 0.9.$$

Step 2 — apply sigmoid:

$$P(\text{Make}) = \text{Sig}(0.9) = \frac{1}{1 + e^{-0.9}} \approx 0.711.$$

71.1% make probability — Dame Time! 🔥

Worked example — forward + one GD step

$\mathbf{x} = (1, 2)^\top$ (with $x_0 = 1$), $\mathbf{w} = (-1, 0.8)^\top$, $y = 1$, $\alpha = 0.1$.

$$z = \mathbf{w}^\top\mathbf{x} = (-1)(1) + (0.8)(2) = 0.6,$$ $$h = \text{Sig}(0.6) = \frac{1}{1 + 0.5488} = 0.6457,$$ $$\text{BCE} = -\log(0.6457) = 0.4373,$$ $$(h - y)\mathbf{x} = (0.6457 - 1)(1, 2) = (-0.3543, -0.7086),$$ $$\mathbf{w}_{\text{new}} = \mathbf{w} - \alpha(h-y)\mathbf{x} = (-0.9646, 0.8709).$$

8. Softmax regression (multinomial logistic)

Softmax $$P(y\!=\!k \mid \mathbf{x}) = \text{softmax}_k(\mathbf{W}\mathbf{x}) = \frac{\exp(\mathbf{w}_k^\top \mathbf{x})}{\sum_{\ell=1}^{K} \exp(\mathbf{w}_\ell^\top\mathbf{x})}.$$
Where: $K$ = number of classes; $\mathbf{w}_k$ = weight vector for class $k$; $\mathbf{W}$ = matrix whose rows are $\mathbf{w}_1^\top, \ldots, \mathbf{w}_K^\top$. The denominator normalizes so probabilities sum to 1.

Output:

$$\mathbf{h}_{\mathbf{W}}(\mathbf{x}) = \text{softmax}(\mathbf{W}\mathbf{x}) = \begin{pmatrix} P(y\!=\!1 \mid \mathbf{x}) \\ \vdots \\ P(y\!=\!K \mid \mathbf{x}) \end{pmatrix} \in \mathbb{R}^K.$$

A vector of $K$ probabilities summing to 1.

Categorical cross entropy (CCE)

CCE loss $$\text{NLL}(\mathbf{W}) = -\sum_{i=1}^{N}\sum_{k=1}^{K} y_k^{(i)} \log \text{softmax}_k(\mathbf{W}\mathbf{x}^{(i)}) = -\sum_i \mathbf{y}^{(i)\top}\log\,\text{softmax}(\mathbf{W}\mathbf{x}^{(i)}).$$

Gradient (very compact):

$$\frac{\partial L}{\partial \mathbf{W}} = \sum_i (\mathbf{s}^{(i)} - \mathbf{y}^{(i)})\mathbf{x}^{(i)\top},$$

where $\mathbf{s}^{(i)} = \text{softmax}(\mathbf{W}\mathbf{x}^{(i)})$ is the predicted probability vector.

Softmax parameter dimensions — frequently confused!

For $K$-class classification with input of $D$ features (bias separate):

$$\underbrace{(K \times 1)}_{\text{output}} = \underbrace{(K \times D)}_{\mathbf{W}} \cdot \underbrace{(D \times 1)}_{\mathbf{x}} + \underbrace{(K \times 1)}_{\mathbf{b}}.$$

So $\mathbf{W}$ has dimension $K \times D$ — NOT $D \times K$! Total parameters: $K \cdot D + K = K(D+1)$.

Example: 3-class with 4 features → $\mathbf{W} \in \mathbb{R}^{3 \times 4}$, $\mathbf{b} \in \mathbb{R}^3$, total = $3 \cdot 4 + 3 = 15$.

🏀 NBA — predicting player position

5 classes (PG/SG/SF/PF/C), 5 features (height, weight, RPG, APG, BPG). Parameter shapes: $\mathbf{W} \in \mathbb{R}^{5\times 5}$, $\mathbf{b} \in \mathbb{R}^5$. Total = $5 \cdot 5 + 5 = 30$.

For Nikola Jokić, suppose the logits computed are $\mathbf{W}\mathbf{x} + \mathbf{b} = (1.2, 0.8, 1.1, 3.5, 4.2)$. Then:

$$\mathbf{s} = \frac{1}{e^{1.2} + e^{0.8} + e^{1.1} + e^{3.5} + e^{4.2}}\left(e^{1.2}, e^{0.8}, e^{1.1}, e^{3.5}, e^{4.2}\right) \approx (0.02, 0.01, 0.02, 0.23, 0.72).$$

Most likely position: Center (72%) — matches reality.

9. One-vs-Rest (OVR) vs Softmax

One-vs-Rest
Train $K$ binary classifiers (each: class $k$ vs the rest). Each has its own independent loss.
Softmax
A single model with $K$ output units. One CCE loss for everything.
Tutorial — OVR vs Softmax true/false
  • "5-class OVR needs 4 logistic regressions" → FALSE. Needs $K = 5$ binary classifiers.
  • "Each logistic in OVR has an independent loss" → TRUE.
  • "Softmax optimizes a single log-likelihood" → TRUE.
  • "Softmax output is always in $[0,1]$" → TRUE (in fact in $(0,1)$, and sums to 1).

Note: OVR is not always better than softmax. Softmax is more consistent because it models all class probabilities jointly.

Worked example — softmax NLL

$\mathbf{x} = (1, 0.8, 0.8)^\top$, target $\mathbf{y} = (1, 0, 0)$, $\mathbf{W} = \begin{pmatrix} 0.3 & 0.9 & 0.8 \\ 0.8 & 0.6 & 0.6 \\ 0.3 & 0.4 & 0.9 \end{pmatrix}$.

Logits: $\mathbf{W}\mathbf{x} = (1.66, 1.76, 1.34)$. Exponentials: $(5.259, 5.812, 3.819)$, sum $14.890$. Softmax: $\mathbf{s} = (0.35, 0.39, 0.26)$. NLL: $-\log(0.35) \approx 1.05$.
