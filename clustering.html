Clustering

Unsupervised learning — finding groups of similar data without labels. K-Means, hierarchical, and the evaluation metrics for both.

1. Introduction

Clustering = unsupervised learning: no labels. Goal: find groups of similar data points. Each cluster typically has a representative (a centroid). Types:

  • Partitional: K-Means, K-Medoids (flat, top-down).
  • Hierarchical: Agglomerative (bottom-up), Divisive (top-down). Output: dendrogram.

2. Distance metrics

Almost every clustering decision begins with: "how do we measure distance?" Here are all the metrics you might see in an exam:

1 · Euclidean (L2 norm) $$d(\mathbf{u}, \mathbf{v}) = \sqrt{\sum_{i=1}^{n}(u_i - v_i)^2} = \|\mathbf{u} - \mathbf{v}\|_2.$$

Default for numeric data, straight-line distance. Squared version $d^2$ is used inside SSE because it avoids the square root.

2 · Manhattan / Rectilinear / Taxicab (L1 norm) $$d(\mathbf{u}, \mathbf{v}) = \sum_{i=1}^{n}|u_i - v_i| = \|\mathbf{u} - \mathbf{v}\|_1.$$

Grid-like settings, robust to outliers, high-dimensional data.

3 · Chebyshev (L$_\infty$ norm) $$d(\mathbf{u}, \mathbf{v}) = \max_i |u_i - v_i| = \|\mathbf{u} - \mathbf{v}\|_\infty.$$

Chessboard king moves, worst-case single-axis distance.

4 · Minkowski (general, parameter $p$) $$d_p(\mathbf{u}, \mathbf{v}) = \left(\sum_{i=1}^{n}|u_i - v_i|^p\right)^{1/p}.$$

$p = 1$ → Manhattan; $p = 2$ → Euclidean; $p \to \infty$ → Chebyshev.

5 · Cosine similarity / distance $$\cos(\mathbf{u}, \mathbf{v}) = \frac{\mathbf{u}^\top\mathbf{v}}{\|\mathbf{u}\|_2\|\mathbf{v}\|_2}, \quad d_{\cos} = 1 - \cos(\mathbf{u}, \mathbf{v}).$$

Text/document vectors where direction matters, not magnitude.

6 · Hamming (for categorical / binary) $$d_H(\mathbf{u}, \mathbf{v}) = \sum_{i=1}^{n} \mathbb{1}[u_i \neq v_i].$$

String comparison, DNA sequences, categorical features.

Which metric for which data?
  • Numeric, no outliers, geometric meaning → Euclidean.
  • Numeric with outliers, or grid-aligned → Manhattan.
  • Categorical / binary → Hamming.
  • Text / high-dim sparse → Cosine.

Exam tip: always check which metric is requested. Results differ drastically!

3. K-Means

Algorithm

  1. Pick $k$ initial centroids.
  2. Assign: each point to the nearest centroid.
  3. Update: recompute centroids = mean of members.
  4. Repeat 2–3 until convergence.
Objective (SSE) $$\text{SSE} = \sum_{j=1}^{k}\sum_{\mathbf{x} \in C_j} \|\mathbf{x} - \mathbf{m}_j\|_2^2.$$
Where: $k$ = number of clusters; $C_j$ = points in cluster $j$; $\mathbf{m}_j$ = centroid (mean) of cluster $j$. Lower SSE = tighter clusters.

Complexity: $\mathcal{O}(t \cdot k \cdot n)$ — considered linear, where $t$ = iterations, $n$ = data points.

Weaknesses

  • Works only for numerical data (mean must be defined). For categorical → K-Modes.
  • User must specify $k$.
  • Sensitive to outliers → use K-Medoids instead.
  • Sensitive to initial centroids (different runs may give different results).

Worked example — K-Means with Manhattan distance

8 points: A1(2,10), A2(2,5), A3(8,4), A4(5,8), A5(7,5), A6(6,4), A7(1,2), A8(4,9). $k = 3$. Initial centroids: $C_1 = A1$, $C_2 = A4$, $C_3 = A7$.

Iter 1: Cluster 1 = {A1}; Cluster 2 = {A3, A4, A5, A6, A8}; Cluster 3 = {A2, A7}. New centroids: $\mathbf{m}_1 = (2,10)$, $\mathbf{m}_2 = (6,6)$, $\mathbf{m}_3 = (1.5, 3.5)$.

Iter 2: A8 moves to Cluster 1. Centroids: $(3, 9.5), (6.5, 5.25), (1.5, 3.5)$.

Iter 3: A4 moves to Cluster 1. Centroids: $(3.67, 9), (7, 4.33), (1.5, 3.5)$.

Iter 4: Converged. Final: $\{A1, A4, A8\}, \{A3, A5, A6\}, \{A2, A7\}$.

4. K-Medoids — outlier resistance

K-Means problem

The mean is sensitive to outliers — one extreme point can drag the centroid far, distorting the cluster.

K-Medoids: instead of using the mean, the centroid is chosen as an actual data point that minimizes total distance to cluster members (multidimensional median analogue). More robust to outliers.

K-MeansK-Medoids

5. Hierarchical (agglomerative) clustering

Idea: start with each point as its own cluster, repeatedly merge the closest pair until one cluster remains. Output: a dendrogram you can cut at any height to get any number of clusters.

6. Linkage methods

When two clusters $A$ and $B$ have just been merged into $(A,B)$, how do we compute the distance to some other cluster $X$?

1 · Single linkage (nearest-neighbor / MIN) $$d((A,B), X) = \min(d(A,X), d(B,X)) = \min_{a \in A \cup B,\, x \in X} d(a, x).$$

Behavior: can produce long, stringy chains (chaining effect). Strength: can find non-elliptical clusters.

2 · Complete linkage (furthest-neighbor / MAX) $$d((A,B), X) = \max(d(A,X), d(B,X)) = \max_{a \in A \cup B,\, x \in X} d(a, x).$$

Behavior: compact, roughly spherical clusters. Weakness: sensitive to outliers.

3 · Average linkage (UPGMA) $$d((A,B), X) = \text{avg}(d(A,X), d(B,X)) = \frac{1}{|A \cup B||X|}\sum_{a \in A \cup B}\sum_{x \in X} d(a, x).$$

Behavior: balanced. Most commonly used in practice.

4 · Centroid linkage $$d((A,B), X) = d(\mathbf{m}_{A \cup B}, \mathbf{m}_X).$$

Can give non-monotonic dendrograms (rare).

5 · Ward's method

Distance = increase in total within-cluster SSE when merging. Minimizes within-cluster variance; tends to produce equally-sized clusters.

Quick decision

Irregular shapes, chains allowedSingle

7. Silhouette coefficient

The most common internal evaluation metric — no ground truth needed.

For each object $o$ $$a(o) = \frac{1}{|C(o)| - 1}\sum_{p \in C(o)} d(o, p),$$ $$b(o) = \min_{C_i \neq C(o)} \left[ \frac{1}{|C_i|}\sum_{p \in C_i} d(o, p) \right],$$ $$S(o) = \begin{cases} 0, & a(o) = 0 \\ \dfrac{b(o) - a(o)}{\max(a(o), b(o))}, & \text{otherwise.} \end{cases}$$
Where: $a(o)$ = average intra-cluster distance (tightness); $b(o)$ = mean distance to closest other cluster (separation); $S(o) \in [-1, 1]$: ≈ 1 good, ≈ 0 borderline, ≈ −1 misclustered.

Overall scores: $S(C_i) = \tfrac{1}{|C_i|}\sum_{o \in C_i} S(o)$, $S(\mathcal{C}) = \tfrac{1}{|\mathcal{D}|}\sum_{o \in \mathcal{D}} S(o)$.

$S(\mathcal{C})$ rangeStructure strength
0.70 – 1.00Strong
0.50 – 0.70Moderate
0.25 – 0.50Weak
≤ 0.25No structure
Reading a silhouette plot

To draw one: compute $S(o)$ for each point, group by cluster, sort each cluster low-to-high, plot horizontal bars (width = $S(o)$), draw a vertical red line at $S(\mathcal{C})$.

The right number of clusters is the one where:

  1. $S(\mathcal{C})$ is highest (red line farthest right).
  2. Many clusters exceed the red line.
  3. Clusters are similarly sized.
  4. Few points have negative silhouette.

vs Elbow: Elbow uses SSE (lower is better, look for elbow). Silhouette uses $S(\mathcal{C})$ (higher is better, look for the peak).

Worked example (1D)

Cluster $A = \{1, 2\}$, $B = \{5, 6\}$. For $o = 1$ in $A$:

$$a(1) = \frac{1}{|A| - 1} d(1, 2) = \frac{1}{1} \cdot 1 = 1.$$ $$b(1) = \frac{1}{|B|}(d(1,5) + d(1,6)) = \frac{1}{2}(4 + 5) = 4.5.$$ $$S(1) = \frac{4.5 - 1}{4.5} \approx 0.778 \;\text{(strong structure)}.$$

8. Elbow method

Plot SSE versus $k$. Pick the $k$ at the elbow where the curve bends sharply.

$$\text{SSQ} = \sum_{o \in C_i,\, c_i \in c} d^2(o, c_i) \quad \text{(lower is better)}.$$

Note: SSE monotonically decreases as $k$ increases (so SSE alone cannot be used to pick $k$ — combine with Elbow / Silhouette).

9. External evaluation (need ground truth)

Compare object pairs against a ground truth $G$. Let $S_C$ = pairs co-clustered by us, $S_G$ = pairs co-grouped in truth.

Pair statusDefinition
TPCo-clustered AND co-grouped (correctly together)
FPCo-clustered BUT NOT co-grouped (wrongly together)
FNNOT co-clustered BUT co-grouped (wrongly apart)
TNNOT co-clustered AND NOT co-grouped (correctly apart)
Pair-based metrics $$\text{Precision} = \frac{TP}{TP + FP}, \quad \text{Recall} = \frac{TP}{TP + FN}, \quad F1 = \frac{2 \cdot P \cdot R}{P + R}.$$ $$\text{Rand Index} = \frac{TP + TN}{TP + FP + FN + TN}, \quad \text{Jaccard} = \frac{TP}{TP + FP + FN}.$$
Which evaluation when?
  • No ground truth + want a number → Silhouette.
  • No ground truth + choosing $k$ → Elbow on SSE or peak of Silhouette.
  • Have ground truth labels → F1 / Rand Index / Jaccard.
  • Want robust-to-chance metric → Adjusted Rand Index (ARI) or NMI.
🏀 NBA — clustering evaluation

Group players into 5 clusters by stats; compare with official positions (PG/SG/SF/PF/C as ground truth). For a small example with 6 pairs: $TP=3, FP=1, FN=1, TN=1$.

$$\text{Precision} = \frac{3}{4} = 0.75, \quad \text{Recall} = \frac{3}{4} = 0.75, \quad F1 = \frac{2(0.75)(0.75)}{1.5} = 0.75.$$

A high F1 means the clustering matches official positions well.
