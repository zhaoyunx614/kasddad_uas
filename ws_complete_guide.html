Clustering — K-Means & Agglomerative & Silhouette

K-Means with Euclidean Distance · Agglomerative with Manhattan + Average Linkage · Silhouette Coefficient

Question 1 — K-Means Clustering (k=2)
1

The Prediction Formula — h(x)

In K-Means, we assign each data point to the nearest centroid by computing the distance. The distance used here is Euclidean Distance:

\[ d(A, B) = \sqrt{(A_x - B_x)^2 + (A_y - B_y)^2} \]
Symbol Legend
\(A, B\)Two points being compared (e.g., a data point and a centroid)
\(A_x, A_y\)Coordinates of point A
\(d(A,B)\)Euclidean distance between points A and B
2

Step 1 — Compute Distance of Each Point to Each Centroid

Given initial centroids cent1 = (1, 5) and cent2 = (3, 1), for each data point calculate the Euclidean distance to both centroids, then assign to the nearest one.

📐

How to fill the dist1 and dist2 columns: For each row, apply the formula above substituting the data point coordinates and each centroid's coordinates. The cluster column gets C1 if dist1 < dist2, else C2.

Example for point A1 = (1, 5):

\[ \text{dist to cent1} = \sqrt{(1-1)^2 + (5-5)^2} = \sqrt{0+0} = 0 \] \[ \text{dist to cent2} = \sqrt{(1-3)^2 + (5-1)^2} = \sqrt{4+16} = \sqrt{20} \approx 4.472 \]

Since 0 < 4.472, point A1 → Cluster 1.

3

Step 2 — Update Centroids (Recalculate as Mean of Cluster Members)

After assigning all points to clusters, recalculate each centroid as the mean of all points in that cluster:

\[ \text{new centroid}_k = \left( \frac{\sum_{i \in C_k} x_i}{|C_k|}, \; \frac{\sum_{i \in C_k} y_i}{|C_k|} \right) \]
Symbol Legend
\(C_k\)Set of points assigned to cluster k
\(|C_k|\)Number of points in cluster k
\(x_i, y_i\)Coordinates of point i in cluster k
🔁

Stopping Criteria: Repeat Steps 1–2 until the centroids no longer change (converge). In the WS, check after each iteration whether the centroid is the same as the previous one.

Question 2 — Agglomerative Clustering (Manhattan + Average Linkage)
1

Distance Matrix — Manhattan Distance

Manhattan distance (also called L1 distance) between two points:

\[ d_{\text{Manhattan}}(A, B) = |A_x - B_x| + |A_y - B_y| \]
Symbol Legend
\(|{}\cdot{}|\)Absolute value (always take the positive result)
\(A_x, B_x\)x-coordinates of points A and B
\(A_y, B_y\)y-coordinates of points A and B

Fill the distance matrix by computing \(d\) for every pair of points. Note: d(A,A) = 0 and the matrix is symmetric \(d(A,B) = d(B,A)\).

⚠️

Important: The WS says "no normalization" — use raw coordinates directly.

2

Merging Steps — Average Linkage

At each step, merge the pair of clusters with the smallest distance. After merging, update all distances to the new cluster using Average Linkage:

\[ d_{\text{avg}}(C_{\text{new}}, C_j) = \frac{\sum_{a \in C_{\text{new}}} \sum_{b \in C_j} d(a, b)}{|C_{\text{new}}| \cdot |C_j|} \]
Symbol Legend
\(C_{\text{new}}\)The newly merged cluster (contains all members of the two merged clusters)
\(C_j\)Any other existing cluster
\(d(a,b)\)Distance between individual point a and individual point b (from original distance matrix)
\(|C|\)Number of members in a cluster
💡

Practical shortcut for average linkage: When merging cluster C1 (with members) and C2 (with members) into C_new, the new distance to any cluster Cj = average of all pairwise distances between members of C_new and Cj.

The WS uses the alphabetical order rule: when distances are equal, prefer the alphabetically-first pair. List cluster members sorted alphabetically.

Question 3 — Silhouette Coefficient
1

Intra-cluster similarity — a(i)

The intra-cluster similarity a(i) is the average distance from point i to all other points in the same cluster:

\[ a(i) = \frac{1}{|C_i| - 1} \sum_{j \in C_i,\; j \neq i} d(i, j) \]
Symbol Legend
\(a(i)\)Average distance from point i to all OTHER points in its own cluster (lower = more cohesive)
\(C_i\)The cluster that point i belongs to
\(|C_i|-1\)Number of other points in the same cluster (subtract 1 to exclude point i itself)
\(d(i,j)\)Euclidean distance between point i and point j

In the WS, for point (13.5, 7.5) in Cluster C, compute distances to each OTHER Cluster C member, then average them.

2

Inter-cluster dissimilarity — b(i)

The inter-cluster dissimilarity b(i) is the average distance from point i to all points in the nearest OTHER cluster:

\[ b(i) = \min_{k \neq C_i} \frac{1}{|C_k|} \sum_{j \in C_k} d(i, j) \]
Symbol Legend
\(b(i)\)Minimum average distance from point i to points in any other cluster (lower means the point is close to a neighboring cluster)
\(\min_{k \neq C_i}\)Take the minimum over all clusters other than the one i belongs to
\(|C_k|\)Number of points in the other cluster k

Compute the average distance from (13.5, 7.5) to all points in Cluster A, then to all points in Cluster B. Take the minimum of these two averages as b(i).

3

Silhouette Score — s(i)

\[ s(i) = \frac{b(i) - a(i)}{\max(a(i),\; b(i))} \]
Symbol Legend
\(s(i)\)Silhouette score for point i. Range: [−1, 1]. Closer to +1 = well-clustered; near 0 = on boundary; negative = possibly mis-clustered
\(b(i)\)Average distance to nearest other cluster (separation)
\(a(i)\)Average distance within own cluster (cohesion)
\(\max(\cdot)\)The larger of a(i) or b(i) — used to normalize the score to [−1,1]
🧮

Worked example from WS: a(13.5, 7.5) = 2.617 (avg dist to C cluster mates), b(13.5, 7.5) = min(avg to A, avg to B). Then s = (b − a) / max(a, b) = 0.096

Naive Bayes & Probability

Prior probability · Likelihood · Posterior (Bayes Theorem) · Naive Bayes Classifier

✓ Perfect Score — 100/100
1

Core Formula — Bayes Theorem

\[ P(Y \mid X) = \frac{P(X \mid Y) \cdot P(Y)}{P(X)} \]
Symbol Legend
\(P(Y \mid X)\)Posterior: probability of class Y given observed data X — this is what we want
\(P(X \mid Y)\)Likelihood: probability of observing data X given class Y
\(P(Y)\)Prior: probability of class Y before seeing any data (from training data counts)
\(P(X)\)Normalizing constant / evidence: probability of observing X regardless of class. This is the same for all classes, so it can be ignored when comparing classes.
2

Q1–Q2 — Conditional Probability (University Students)

Given: 800 students total, 416 female. 5% of all students take CS. 35% take Business. The rest take Psychology.

📊

How to derive counts from percentages:
Total CS students = 800 × 5% = 40
Total Business students = 800 × 35% = 280
Total Psychology = 800 − 40 − 280 = 480

For conditional probability questions like "P(Female | CS)", use the given information: 2% of female CS students (from Q2) → number of female CS = 120 × 2% × ... Apply definition:

\[ P(A \mid B) = \frac{P(A \cap B)}{P(B)} = \frac{\text{count}(A \text{ and } B)}{\text{count}(B)} \]
3

Q3–Q4 — Total Probability & Bayes (Pipe Factory)

Three factories X, Y, Z produce 2000, 3000, 5000 units respectively. Defect rates: X=3%, Y=4%, Z=2%.

Step A — Prior Probabilities

\[ P(X) = \frac{2000}{10000} = 0.2, \quad P(Y) = \frac{3000}{10000} = 0.3, \quad P(Z) = \frac{5000}{10000} = 0.5 \]

Step B — Total Probability of Defect P(D)

\[ P(D) = P(D|X)\cdot P(X) + P(D|Y)\cdot P(Y) + P(D|Z)\cdot P(Z) \] \[ = 0.03 \times 0.2 + 0.04 \times 0.3 + 0.02 \times 0.5 = 0.006 + 0.012 + 0.01 = 0.028 \]
Symbol Legend
\(P(D|X)\)Probability a pipe is defective given it came from factory X (= defect rate = 0.03)
\(P(D)\)Total probability of a randomly selected pipe being defective (sum over all factories)

Step C — Bayes: P(Y | D)

\[ P(Y \mid D) = \frac{P(D|Y) \cdot P(Y)}{P(D)} = \frac{0.04 \times 0.3}{0.028} = \frac{0.012}{0.028} \approx 0.429 \]
4

Q5 — Naive Bayes Classifier (Energy Usage)

Naive Bayes classifies a new data point by computing the unnormalized posterior for each class, then normalizing.

Step A — Prior

\[ P(c) = \frac{N_c}{N} \]
Symbol Legend
\(P(c)\)Prior probability of class c (e.g., Low or High)
\(N_c\)Number of training samples belonging to class c
\(N\)Total number of training samples

Step B — Likelihood (Class Conditional Density) for Categorical Features

\[ P(x_d = v \mid y = c) = \frac{\text{count}(x_d = v \text{ and } y = c)}{N_c} \]
Symbol Legend
\(x_d\)Feature d (e.g., Solar Panels or Energy-Efficient Appliances)
\(v\)The specific value of feature d (e.g., Yes or No)
\(y = c\)The class label (Low or High energy usage)
\(N_c\)Number of training samples with class c (denominator = class count)

Step C — Unnormalized Posterior (Naive: assume feature independence)

\[ P(c \mid \mathbf{x}) \propto P(c) \cdot \prod_{d=1}^{D} P(x_d \mid y = c) \]
Symbol Legend
\(\propto\)"Proportional to" — we skip dividing by P(x) since it's the same for all classes
\(\prod\)Product (multiply together) of all per-feature likelihoods
\(D\)Number of features

Step D — Normalize to get true posterior

\[ P(c \mid \mathbf{x}) = \frac{P(c) \cdot \prod_d P(x_d \mid c)}{\sum_{c'} P(c') \cdot \prod_d P(x_d \mid c')} \]

Sum all unnormalized posteriors across classes, then divide each by that sum.

Question 1 — FFNN Architecture & Parameter Count
1

Counting Parameters in a Layer

A layer with \(n_{\text{in}}\) inputs and \(n_{\text{out}}\) outputs has:

\[ \text{Parameters in layer} = (n_{\text{in}} \times n_{\text{out}}) + n_{\text{out}} \]
Symbol Legend
\(n_{\text{in}}\)Number of input features (or units from previous layer)
\(n_{\text{out}}\)Number of units (neurons) in this layer
\(n_{\text{in}} \times n_{\text{out}}\)Weight matrix W — one weight per (input, output) pair
\(+\; n_{\text{out}}\)Bias vector b — one bias per output unit

For 13 inputs → 23 units (Layer 1): params = (13×23)+23 = 299+23 = 322

For 23 inputs → 3 units (Layer 2): params = (23×3)+3 = 69+3 = 72

Total = 322 + 72 = 394 parameters

🧠

Activation function for 3-class output: Use Softmax, which outputs a probability distribution over K classes (all outputs sum to 1). For hidden layers with arbitrary outputs, Sigmoid or ReLU are used; for multi-class output → Softmax.

Question 2 — Full Forward & Backward Pass
1

Forward Pass — Layer 1 (Softmax Activation)

Given input \(\mathbf{x} = (-1, 1)\), the forward pass through Layer 1:

\[ \mathbf{p}^{(1)} = W^{(1)}\mathbf{x} + \mathbf{b}^{(1)} \quad \text{(pre-activation)} \]
\[ \mathbf{a}^{(1)} = \text{softmax}(\mathbf{p}^{(1)}) \quad \text{(activation)} \]
Symbol Legend
\(W^{(1)}\)Weight matrix of Layer 1 (shape: units × inputs)
\(\mathbf{b}^{(1)}\)Bias vector of Layer 1
\(\mathbf{p}^{(1)}\)Pre-activation vector (raw linear combination before applying activation)
\(\mathbf{a}^{(1)}\)Activation vector (output of Layer 1 after applying softmax)

Softmax Formula

\[ \text{softmax}(u_k) = \frac{e^{u_k}}{\sum_{j} e^{u_j}} \]
Symbol Legend
\(u_k\)The k-th element of the pre-activation vector
\(e^{u_k}\)Exponential of the k-th element
\(\sum_j e^{u_j}\)Sum of exponentials of ALL elements — this normalizes the output so they sum to 1

Example: if \(\mathbf{p}^{(1)} = (1.9, 0.5)\), then softmax: \(e^{1.9} \approx 6.686\), \(e^{0.5} \approx 1.649\), sum = 8.335, so softmax = (6.686/8.335, 1.649/8.335) ≈ (0.802, 0.198). But the WS gives \(a_1^{(1)} = 2.039\) — this is for a layer NOT using softmax on a 2-unit hidden layer; those are from the matrix multiply + bias directly with softmax applied per component.

2

Forward Pass — Layer 2 & Loss

\[ \mathbf{p}^{(2)} = W^{(2)}\mathbf{a}^{(1)} + \mathbf{b}^{(2)} \]
\[ \mathbf{a}^{(2)} = \text{softmax}(\mathbf{p}^{(2)}) \]

Cross-Entropy Loss (for 3-class, target y = class 1)

\[ L = -\sum_{k=1}^{K} y_k \log(a_k^{(2)}) = -\log(a_{\text{target}}^{(2)}) \]
Symbol Legend
\(K\)Number of classes (K=3 here)
\(y_k\)One-hot target: 1 for the correct class, 0 for all others
\(a_k^{(2)}\)Predicted probability for class k (output of softmax in Layer 2)
\(\log\)Natural logarithm (ln)
\(-\log(\cdot)\)The loss is large when the predicted probability for the true class is small
📌

Since y is one-hot, only the term for the correct class survives. Target = class 1 (index 0) → \(L = -\log(a_0^{(2)})\)

3

Backward Pass — Gradients at Output Layer (Softmax + Cross-Entropy)

The combined derivative of softmax + cross-entropy loss with respect to the pre-activation \(\mathbf{p}^{(2)}\) simplifies beautifully:

\[ \bar{\mathbf{p}}^{(2)} = \frac{\partial L}{\partial \mathbf{p}^{(2)}} = \mathbf{a}^{(2)} - \mathbf{y} \]
Symbol Legend
\(\bar{\mathbf{p}}^{(2)}\)Gradient of loss w.r.t. pre-activation of Layer 2 (a vector)
\(\mathbf{a}^{(2)}\)Predicted probability vector from forward pass
\(\mathbf{y}\)One-hot target vector

Example: if \(\mathbf{a}^{(2)} = (0.255, 0.513, 0.232)\) and target is class 0 (y=(1,0,0)), then \(\bar{\mathbf{p}}^{(2)} = (0.255-1, 0.513-0, 0.232-0) = (-0.745, 0.513, 0.232)\).

4

Backward Pass — Gradients for W and b

\[ \bar{W}^{(2)} = \frac{\partial L}{\partial W^{(2)}} = \bar{\mathbf{p}}^{(2)} \cdot (\mathbf{a}^{(1)})^T \]
\[ \bar{\mathbf{b}}^{(2)} = \frac{\partial L}{\partial \mathbf{b}^{(2)}} = \bar{\mathbf{p}}^{(2)} \]
\[ \bar{\mathbf{a}}^{(1)} = \frac{\partial L}{\partial \mathbf{a}^{(1)}} = (W^{(2)})^T \cdot \bar{\mathbf{p}}^{(2)} \]
Symbol Legend
\(\bar{W}^{(2)}\)Gradient of loss w.r.t. weight matrix of Layer 2 — same shape as \(W^{(2)}\)
\(\bar{\mathbf{p}}^{(2)} \cdot (\mathbf{a}^{(1)})^T\)Outer product: column vector × row vector = matrix
\(\bar{\mathbf{b}}^{(2)}\)Gradient of loss w.r.t. bias of Layer 2 = same as \(\bar{\mathbf{p}}^{(2)}\)
\(\bar{\mathbf{a}}^{(1)}\)Gradient flowing back to Layer 1's activations

Then propagate back through Layer 1's softmax activation using the Jacobian of softmax, then compute \(\bar{W}^{(1)}\) and \(\bar{\mathbf{b}}^{(1)}\) similarly.

5

Weight Update — Gradient Descent

\[ W^{(l)}_{\text{new}} = W^{(l)} - \alpha \cdot \bar{W}^{(l)} \]
\[ \mathbf{b}^{(l)}_{\text{new}} = \mathbf{b}^{(l)} - \alpha \cdot \bar{\mathbf{b}}^{(l)} \]
Symbol Legend
\(\alpha\)Learning rate (given in the WS, e.g., α = 0.1)
\(W^{(l)}\)Current weight matrix of layer l
\(\bar{W}^{(l)}\)Gradient of loss w.r.t. W^(l) computed in backward pass
\(W^{(l)}_{\text{new}}\)Updated weight matrix after one gradient descent step
🔢

Apply element-wise: each weight \(w_{ij}^{(l)}\) is updated as \(w_{ij}^{(l)} - \alpha \cdot \bar{w}_{ij}^{(l)}\)

Key Formulas — What Everything Means

Prediction Model h(x)

\[ h(\mathbf{x}) = w_0 x_0 + w_1 x_1 + w_2 x_2 = \mathbf{w}^T \mathbf{x} \]
Symbol Legend
\(h(\mathbf{x})\)Model prediction for input x
\(w_0\)Bias weight (multiplied by \(x_0 = 1\) always)
\(x_0\)Always = 1 (bias/intercept term — a constant added to the feature vector)
\(w_1, w_2\)Weights for features \(x_1\) and \(x_2\)
\(\mathbf{w}^T \mathbf{x}\)Dot product of weight vector and augmented input vector

Gradient (per data point i) — Linear Regression

\[ \nabla_w L_i = (h(\mathbf{x}^{(i)}) - y^{(i)}) \cdot \mathbf{x}^{(i)T} \]
Symbol Legend
\((h(\mathbf{x}^{(i)}) - y^{(i)})\)Prediction error for sample i: predicted value minus true value
\(\mathbf{x}^{(i)T}\)The input row for sample i as a row vector: \((x_0^{(i)}, x_1^{(i)}, x_2^{(i)})\)
\(\nabla_w L_i\)Gradient vector for sample i — has one component per weight

Weight Update — Gradient Descent

\[ \mathbf{w}_{\text{new}} = \mathbf{w} - \frac{\alpha}{N_{\text{batch}}} \sum_{i \in \text{batch}} \nabla_w L_i \]
Symbol Legend
\(\alpha\)Learning rate (given as 0.05 in all questions)
\(N_{\text{batch}}\)Number of samples in the batch (for batch GD = all 6; for minibatch = 2)
\(\sum_{i \in \text{batch}} \nabla_w L_i\)Sum of gradients over all samples in the current batch

Question 3 — Logistic Regression (Binary Classification)
1

Sigmoid Function — Replacing h(x) for Classification

For logistic regression, the prediction uses the sigmoid function applied to the linear combination:

\[ h(\mathbf{x}) = \sigma(\mathbf{w}^T\mathbf{x}) = \frac{1}{1 + e^{-\mathbf{w}^T\mathbf{x}}} \]
Symbol Legend
\(\sigma(\cdot)\)Sigmoid function — maps any real number to (0,1), representing a probability
\(e^{-(\cdot)}\)Exponential of the negated linear combination
\(h(\mathbf{x})\)Predicted probability that sample x belongs to class 1

Example for sample 1 (x₁=1, x₂=2), w=(1,1,−1):

\[ \mathbf{w}^T\mathbf{x} = 1\cdot1 + 1\cdot1 + (-1)\cdot2 = 1 + 1 - 2 = 0 \] \[ h(x) = \sigma(0) = \frac{1}{1+e^0} = \frac{1}{1+1} = 0.5 \]
2

Gradient for Logistic Regression — Same Form as Linear!

\[ \nabla_w L_i^{\text{logistic}} = (\sigma(\mathbf{w}^T\mathbf{x}^{(i)}) - y^{(i)}) \cdot \mathbf{x}^{(i)T} \]
✅

Great news: The gradient formula for logistic regression (BCE loss) has the same structure as linear regression (SSE loss)! Just replace \(h(x)\) with \(\sigma(\mathbf{w}^T\mathbf{x})\). All table columns are computed identically.
